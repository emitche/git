What is version control?

Imagine you are designing a logo. You start out with some image files and the main image file you are designing.

mopsie1.png

Let's say you reach a point in your design where you want to try out something new without impacting the old file. So you save a copy. 

awesome-logo.xcf
awesome-logo-2.xcf
This can quickly become a bit of a tangle. When you are working on a project (building a site, an application, a logo, a book), it's messy and hard to keep track when you save many files or many folders of the same project. Version control monitors the changes you make in a project and keeps record of your changes. That way, instead of saving multiple file copies, you instead are saving your changes, making record of why you made those changes.
